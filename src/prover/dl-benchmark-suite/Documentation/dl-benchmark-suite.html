The Description Logic Benchmark Suite

This DL Benchmark Suite is based on benchmark problems for the DL98 System Comparison with addtional ABox benchmarks and extended application knowledge bases. In addition, the TABLEAUX/TANCS'99 benchmark problems are integrated. Many people have contributed to this collection (see the acknowledgements).

With contributions from Volker Haarslev, Ian Horrocks, Ralf Möller and Peter Patel-Schneider.

See also the DL'99 Systems Comparison at the 1999 International Workshop on Description Logics - DL'99

If you have additional problem sets to be included into the benchmark suite, please send a message.

New benchmarks compared to the DL98 System Comparison

  1. New application knowledge bases (e.g. TBoxes with GCIs).
  2. Additional and more demanding ABox tests.
  3. Problems of (un)satisfiability in modal logic K or, in equivalent terms, of concept satisfiability in the description logic ALC (with only one role) and empty TBoxes.
  4. Problems that are still in PSPACE but require the use of global axioms when encoded in modal logic.

More demanding ABox benchmarks

The initial DL'98 ABox benchmarks are given as a set of inference problems consisting of a TBox, an ABox and a set of instance checking queries. However, for optimized systems such as RACE, these DL'98 ABox benchmarks are far too simple, i.e. the execution time is almost not measurable. Even for the hardest problems the runtimes are below 3 seconds (realization inclusive). Therefore, we have compiled a new set of benchmarks for ALC ABoxes based on more complex TBoxes as before (based on the TBoxes of the DL'98 System Comparison). Each benchmark now consists of a set of subproblems which, in turn, consist of an ALC TBox, an ALC ABox and a set of instance checking problems which should evaluate to true. We generated a set of nine new ABox benchmark problem sets, each with up to 21 problems. The current versions of FaCT and DLP do not support ABox reasoning. Thus, only RACE can be used to run the ABox tests.

More demanding application knowledge bases

With respect to the DL'98 System Comparison new application knowledge bases have been added. The DL'98 versions did not use role hierachies and domain and range restrictions for roles. The new application knowledge bases can be run with RACE. FaCT 1.11 does not support number restrictions. DLP cannot handle GCIs.
